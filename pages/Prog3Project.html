Down We Go

Context

This project was done for my programming 3 class, during my 2nd semester, in cooperation with another student. At that point I had a lot more knowledge, including lists, classes, WPF for display and a remake of the "engine" of my programming 1 project. Unfortunately, this project had it's own share of problems, mainly because it was a team project. Me and my teammate split the project in two, with me doing the "backend" (game logic, saving, etc.) and them doing the "frontend" (screens, screen design, etc.). However, there were some issues with connecting the front and back end, so I took over a part of the front end, bringing the project to completion.

Description

In a lot of ways, this is the next version of the programming 1 project. It's based on the improved engine of that project, although the enemies are not moving, and there's no ranged combat. The main inspiration for it was Pixel Dungeon, both for the style and gameplay. Instead, because of the requirement for the number of screens, we made the combat a separate screen, along with an inventory system. While WPF is not the idea solution for games, it's still better than my previous terminal implementation. Overall, it's far better than my previous project, both in look and feel, and showcases my progress in programming.

Demonstration

The title screen. Allows you to view instructions or load an existing game.

The instructions for playing the game.

The main screen/map. From here, you move around, pick up items, fight enemies and reach the exit to win.

The inventory screen, where you can view and use the items you picked up. You can click on items to see what they do.

The combat screen, where you fight an enemy you ran into on the map. Here, you can lose if you run out of health.
